1. Density curves

Task:

Create density curves (geom_density()) for the relative lignin content (rel_lignin). The fill of the curve should correspond to their type (mutant or WT) and be semi-transparent (alpha = 0.5). Lastly, get rid of the hideous grey background by applying our custom theme_sulis().

Bonus:

Check the arguments of coord_cartesian() to figure out how to remove the scale expansion (i.e. the little gap between 0 and the axis line).

2. Violin and boxplots

Task:

Plot the distribution of predicted carbohydrate/lignin ratios (rel_CL) as boxplots (geom_boxplot()) on top of violin plots (geom_violin()), grouped by the number mutated genes (target_genes). Reduce the width of the boxplots to fit within the violin plots (check ?geom_boxplot() for the right argument). Fill the violin plots according to the number of mutated genes, using the continuous viridis colour scale (scale_colour_viridis_c()), which is designed to be interpretable by viewers with common forms of colour blindness.

Data:

"data/Sulis2023_predictions.tsv"

Hint:

If you want to use a continuous variable as a grouping factor on a discrete x-axis, you might have to wrap it inside as.factor().

data <- read_tsv("data/Sulis2023_predictions.tsv")
## Rows: 69124 Columns: 5
##  Column specification 
## Delimiter: "\t"
## chr (1): strategies
## dbl (4): target_genes, rel_lignin, rel_CL, rel_volume
## 
##  Use `spec()` to retrieve the full column specification for this data.
##  Specify the column types or set `show_col_types = FALSE` to quiet this message.
ggplot(
  data = data,
  aes(
    x = as.factor(target_genes),
    y = rel_CL
  )
) +
  geom_violin(aes(fill = target_genes)) +
  geom_boxplot(width = 0.05) +
  scale_fill_viridis_c() +
  theme_sulis()

3. 2D density and facets

Task:

Using the data of mutant predictions, plot the relative stem volume (rel_volume) against relative lignin content (rel_lignin). You can start by trying geom_point(), but youll see that the enormous number of points creates a black mass that is difficult to interpret. Instead, try geom_density2d_filled() to get a 2D density map of the predictions.

This looks OK, but what if you want to compare those densities relative to the number of targeted genes? facet_wrap() lets you split your figure into smaller facets, each representing a subgroup of the data. Use facet_wrap() to split your plot by target_genes. Lastly, add white dashed lines at 100% of x and y axes for orientation (check ?geom_abline()).

Bonus:

Seven facets look a bit inelegant, and the 0 facet (i.e. WT) is useless anyways. Going a bit beyond the scope of this workshop, replace your data argument with data = data |> filter(target_genes > 0) to remove the 0 facet (assuming you named your data frame data).

Also, perhaps give it another colour scheme?

Data:

"data/Sulis2023_predictions.tsv"

Hint:

The unequal number of data points in the different facets makes the densities in facets with fewer target genes invisible. To calculate the bins independently in each facet, set the geom_density2d_filled() option contour_var to "ndensity".

data <- read_tsv("data/Sulis2023_predictions.tsv")
## Rows: 69124 Columns: 5
##  Column specification 
## Delimiter: "\t"
## chr (1): strategies
## dbl (4): target_genes, rel_lignin, rel_CL, rel_volume
## 
##  Use `spec()` to retrieve the full column specification for this data.
##  Specify the column types or set `show_col_types = FALSE` to quiet this message.
ggplot(
  data = data |> filter(target_genes > 0),
  aes(
    x = rel_lignin,
    y = rel_volume
  )
) +
  geom_density2d_filled(contour_var = "ndensity") +
  scale_fill_viridis_d(option = "B") +
  geom_hline(yintercept = 100, colour = "white", linetype = "dashed") +
  geom_vline(xintercept = 100, colour = "white", linetype = "dashed") +
  theme_sulis() +
  facet_wrap(~target_genes)
